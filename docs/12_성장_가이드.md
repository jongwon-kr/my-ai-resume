# CloneCV 성장 가이드

소프트 런치([11_소프트_런치_가이드.md](./11_소프트_런치_가이드.md))에서 긍정 신호가 나온 뒤 사용자를 확대하는 방법입니다.

## 내장 성장 루프

| 장치 | 위치 | 역할 |
|------|------|------|
| 워터마크 | 공개 프로필 하단 | 방문자 → CloneCV 유입 |
| 카카오/X 공유 | 공개 프로필 | 발행자가 링크 확산 |
| @kimdev 예시 | 랜딩 | 체험 → 가입 전환 |

## 채널별 액션

### 1. 콘텐츠 (Velog / 브런치)

- **제목 예:** "PDF 이력서 대신 AI와 대화하는 포트폴리오를 만들었다"
- 본문: 문제(정적 PDF) → CloneCV 데모 `@kimdev` → 본인 `@slug` 스크린샷
- CTA: 랜딩 `무료로 시작하기`

### 2. 커뮤니티 (Show & Tell)

- OKKY, 디스코드 개발 채널 — **과도한 홍보 금지**, 만든 과정·기술 회고 형태
- "피드백 받고 싶다"는 톤이 효과적

### 3. LinkedIn / X

- 발행 후 **공유 버튼** 스크린 + 30초 데모 GIF
- 해시태그: `#포트폴리오` `#이력서` `#개발자` (과다 사용 지양)

### 4. 지원 맥락

- "이력서 PDF + CloneCV 링크"를 함께 제출하는 사용자 스토리 수집 → 랜딩 사례 섹션 후보

## 측정 (주간)

```
가입 → 온보딩 완료 → 첫 발행 → 첫 방문자 채팅
```

Supabase SQL 또는 Dashboard로 퍼널 추적.  
추후 PostHog/Vercel Analytics 연동 시 동일 이벤트명 사용 권장.

## 비용 관리

- Gemini: 채팅 rate limit(5/min, 50/day) 유지, 이상 트래픽 시 `/api/prompt/generate` limit 추가
- Upstash: 무료 tier 한도 모니터링
- Supabase: Storage(avatars), DB row growth

## 다음 기능 우선순위 (트래픽 증가 시)

1. 발행 API rate limit
2. 프로필 삭제 UI (API: `DELETE /api/profiles/[profileId]`)
3. CAPTCHA on inquiries (스팸 지속 시)
4. OG 한글 폰트
5. Product Hunt (인프라·비용 검증 후)
