버전: v1.0 (MVP)
작성일: 2026-07-05

---

## 1. 프로젝트 개요

| 항목           | 내용                                                                                                  |
| -------------- | ----------------------------------------------------------------------------------------------------- |
| 서비스명(가칭) | CloneCV                                                                                               |
| 한 줄 정의     | 이력서 대신 나와 동기화된 AI 챗봇 링크를 제출하여, 면접관이 대화로 지원자를 사전 검증하게 하는 서비스 |
| 타깃 고객      | IT 취업 준비생, 주니어~미들급 이직자                                                                  |
| 핵심 가치 제안 | "텍스트 이력서"가 아닌 "대화형 검증"으로 기술적 깊이를 어필                                           |
| MVP 목표 기간  | 1개월                                                                                                 |
| MVP 목표       | 완전 무료, 바이럴을 통한 트래픽 확보                                                                  |

기존 기획서의 핵심 아이디어는 그대로 유지하되, 아래에서 원본 기획서에 **명시되지 않았거나 모호했던 부분**을 보완했습니다. (⚠️ 표시는 원본 기획서 대비 보완/추가된 항목)

## 2. 이해관계자 및 사용자 유형

| 유형                         | 설명                                    | 권한                                         |
| ---------------------------- | --------------------------------------- | -------------------------------------------- |
| 지원자 (Owner)               | 프로필을 생성하고 소유하는 회원         | 본인 프로필 CRUD, 대화 로그 열람             |
| 방문자 (Interviewer/Visitor) | 공유 링크로 접속해 AI와 대화하는 비회원 | 프로필 열람, 챗봇과 대화만 가능 (비로그인)   |
| 관리자 (Admin)               | ⚠️ **[추가]** 서비스 운영자             | 신고 처리, 악성 콘텐츠 삭제, 사용량 모니터링 |

⚠️ **[중요 보완]** 원본 기획서에는 회원가입/인증 흐름이 전혀 언급되지 않았습니다. 지원자는 "본인 프로필 소유"가 핵심 가치이므로 **최소한의 인증(이메일 또는 소셜 로그인)** 이 MVP에 반드시 포함되어야 합니다. (2번 항목 참고)

## 3. 기능 요구사항 (Functional Requirements)

### FR-1. 인증 및 계정 관리 ⚠️ [신규 추가 — MUST]

- FR-1.1 이메일/비밀번호 또는 소셜 로그인(Google 우선)으로 회원가입/로그인
- FR-1.2 로그인한 유저만 본인 프로필 생성/수정 가능
- FR-1.3 고유 ID(슬러그) 설정 — 중복 체크, `clonecv.com/@ID` 형태로 사용
- FR-1.4 비밀번호 재설정 (이메일 기반)

> 근거: Public Profile이 `/@ID` 링크로 존재하려면 "누가 이 ID를 소유하는가"가 시스템적으로 보장되어야 함. 인증 없이는 ID 선점/도용 문제가 즉시 발생.

### FR-2. 정형 데이터 입력 폼 (Resume Builder)

- FR-2.1 멀티스텝 폼 (기본정보 → 기술스택 → 프로젝트 3개 → 트러블슈팅)
- FR-2.2 임시저장 (Draft) 기능 ⚠️ — 폼이 여러 단계이므로 중간 이탈 시 데이터 유실 방지 필수
- FR-2.3 프로젝트 항목: 프로젝트명, 기간, 역할, 사용 기술, 담당 업무(자유 텍스트), 성과(정량/정성), 트러블슈팅 경험(상황-문제-해결-결과 구조 권장)
- FR-2.4 입력 글자수 제한 및 가이드 툴팁 (좋은 답변 예시 제공) ⚠️
- FR-2.5 제출 완료 시 시스템 프롬프트 자동 생성 트리거

### FR-3. 퍼블릭 공유 프로필 페이지

- FR-3.1 `clonecv.com/@ID` 형태의 공개 페이지
- FR-3.2 좌측: 자동 생성된 웹 이력서 UI (반응형)
- FR-3.3 우측: AI 챗봇 대화 UI
- FR-3.4 모바일 대응 — 좌우 분할이 아닌 탭/스택 레이아웃으로 전환 ⚠️
- FR-3.5 비공개 설정 (프로필 자체를 비공개로 전환 가능) ⚠️ — 구직 활동 노출을 꺼리는 유저 배려
- FR-3.6 프로필 조회수 카운트 (소유자만 확인 가능) ⚠️ — 바이럴 효과 체감 지표

### FR-4. RAG 기반 AI 대화 엔진

- FR-4.1 시스템 프롬프트: 유저의 정형 데이터를 조합해 페르소나 프롬프트 생성
- FR-4.2 범위 밖 질문 방어 로직 (연봉, 개인 신상, 회사 기밀 등 질문 시 정해진 응답으로 회피)
- FR-4.3 1인칭 시점 자연스러운 답변 생성 (LLM 호출)
- FR-4.4 대화 히스토리 유지 (세션 내 컨텍스트 유지) ⚠️ — 원본에 언급 없었으나 자연스러운 대화를 위해 필수
- FR-4.5 응답 스트리밍 (타이핑 효과) ⚠️ — 체감 품질 향상, 챗봇 UX 핵심
- FR-4.6 대화 로그 저장 및 소유자에게 열람 제공 ⚠️ — "면접관이 무엇을 물어봤는지" 확인은 지원자 입장에서 강력한 가치
- FR-4.7 악성/공격적 질문에 대한 콘텐츠 필터링 ⚠️ — 공개 링크 특성상 어뷰징 방지 필수

### FR-5. 성장/바이럴 장치

- FR-5.1 프로필 하단 워터마크 CTA 버튼
- FR-5.2 소셜 공유 버튼 (카카오톡, 링크 복사, X/트위터) ⚠️
- FR-5.3 OG 태그/메타 이미지 자동 생성 (링크 공유 시 미리보기) ⚠️ — 카톡방/디스콰이엇 공유 시 클릭률에 직결되는 필수 요소
- FR-5.4 "압박 면접 챌린지" 유도 문구 등 게임화 요소는 랜딩페이지 카피로 반영

### FR-6. 레이트 리미팅 및 어뷰징 방지 ⚠️ [신규 추가 — MUST]

- FR-6.1 IP/세션 기준 시간당 메시지 전송 횟수 제한
- FR-6.2 유저별 일일 LLM 호출 총량 제한 (비용 방어)
- FR-6.3 신고하기 버튼 (부적절한 프로필/대화 신고)

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 구분      | 요구사항                                                                                           |
| --------- | -------------------------------------------------------------------------------------------------- |
| 성능      | 챗봇 첫 토큰 응답 3초 이내(TTFB), 스트리밍으로 체감 대기시간 최소화                                |
| 보안      | 개인정보(이름/연락처 등) 암호화 저장, 프로필 소유자만 수정 가능하도록 RLS(Row Level Security) 적용 |
| 비용관리  | LLM API 호출 비용이 서비스 성패를 좌우 — 캐싱, 레이트리밋, 토큰 길이 제한 필수                     |
| 가용성    | 바이럴 트래픽 스파이크 대응 가능한 서버리스/오토스케일 구조                                        |
| 확장성    | 추후 유료화, PDF 파싱, 음성 대화 추가를 고려한 모듈형 설계                                         |
| 접근성    | 모바일 브라우저에서 카톡 공유 링크 클릭 시 즉시 정상 렌더링                                        |
| 법적/윤리 | AI가 실제 채용 결정처럼 오인되지 않도록 "AI 클론이며 참고용" 명시 문구 상시 노출 ⚠️                |

---

## 5. Out of Scope (MVP 제외 — 원본 유지 + 보완)

- PDF 자동 업로드 및 파싱
- 유료 결제 연동
- 음성 대화 (TTS/STT)
- ⚠️ 다국어 지원 (한국어 우선, 영어는 v2)
- ⚠️ 팀/기업용 대시보드 (채용담당자가 여러 지원자 비교하는 기능은 v2)
- ⚠️ 면접관 피드백 수집 기능 (v2)
