# CloneCV 개발 과정 기록 가이드

버전: v1.0 | 작성일: 2026-07-09

CloneCV 개발 과정을 **사실 기반**으로 남기기 위한 가이드입니다.  
초안은 `docs/blog/`에 두고, Tistory·Velog 등 외부 발행은 **수동**으로 합니다.

에이전트: `.cursor/skills/blog-writer/SKILL.md`

---

## 1. 파일 규칙

| 항목 | 규칙 |
|------|------|
| 위치 | `docs/blog/` |
| 이름 | `YYYY-MM-DD_주제.md` 또는 `NN_초안_주제.md` |
| 언어 | 한국어 |
| 길이 | 본문 800~1500자 권장 (길면 LEARN만 남기고 축약) |

가이드 파일(`00_작성_가이드.md`)은 초안이 아닙니다.

---

## 2. 글 구조 (필수)

1. **HOOK** — 한 문단. 왜 이 작업을 했는지 / 무엇이 막혔는지
2. **STORY** — 무엇을 바꿨는지 (기능·커밋·결정). 과장 없이
3. **LEARN** — 다음에 쓸 교훈 3개 이내
4. **NEXT** — 다음 할 일 1~3개 + (선택) repo/Live 링크

스크린샷이 있으면 `docs/screenshots/` 경로를 적고, 외부 플랫폼에는 직접 업로드합니다.

---

## 3. 금지

- 회사명·직장·개인 식별 정보·사적 환경(노트북/집 등)
- 과장된 성과·미구현 기능을 완료처럼 서술
- 제품 코드 대규모 수정 (블로그 에이전트는 문서만)
- 예전 Tistory 사적 초안 복구·재사용

---

## 4. 재료

우선순위:

1. `docs/08_개발일지.md` 최근 항목
2. `git log` / 관련 커밋 메시지
3. `docs/07_현황감사.md` backlog·기능 표
4. README 화면 미리보기 절

---

## 5. 호출 예시

```
@docs/blog/00_작성_가이드.md
@docs/08_개발일지.md

이번 주 작업을 블로그 초안으로 작성해줘.
주제: [한 줄]
docs/blog/ 아래에 저장.
```

---

## 6. 초안 체크리스트

- [ ] HOOK → STORY → LEARN → NEXT
- [ ] 사적·식별 정보 없음
- [ ] 구현된 내용만 서술
- [ ] 파일명이 규칙에 맞음
