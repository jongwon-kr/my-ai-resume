# CloneCV 프로덕션 출시 체크리스트

소프트 런치 전 Vercel·Supabase에서 아래 항목을 확인합니다.

## 1. 환경 변수 (Vercel)

| 변수 | 필수 | 용도 |
|------|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Supabase 프로젝트 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | 클라이언트·RLS API |
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ | 공개 프로필 조회, 채팅 DB, 문의/신고 |
| `GEMINI_API_KEY` | ✅ | 채팅, PDF import, 프롬프트 생성 |
| `UPSTASH_REDIS_REST_URL` | ✅ | 채팅·신고·문의·PDF import 레이트리밋 (없으면 prod 500) |
| `UPSTASH_REDIS_REST_TOKEN` | ✅ | Upstash 인증 |
| `NEXT_PUBLIC_SITE_URL` | ✅ | OG·리다이렉트 (예: `https://my-ai-resume-alpha.vercel.app`) |
| `NEXT_PUBLIC_KAKAO_JS_KEY` | 권장 | 카카오톡 공유 |
| `RESEND_API_KEY` | 선택 | 문의 이메일 알림 |
| `INQUIRY_FROM_EMAIL` | 선택 | Resend 발신 주소 |
| `SENTRY_DSN` | 권장 | 에러 모니터링 |
| `SENTRY_AUTH_TOKEN` | 선택 | Sentry 소스맵 업로드 (CI) |

로컬 템플릿: [`.env.local.example`](../.env.local.example)

## 2. 데이터베이스

```bash
npm run db:push
```

마이그레이션 **16개**가 원격 Supabase에 적용되어야 합니다.  
특히 다중 프로필·라벨:

- `20260714100000_multi_profiles.sql`
- `20260714110000_profile_label.sql`
- `20260715100000_mvp_security_hardening.sql`

적용 후 Supabase Dashboard → Database → Migrations에서 최신 파일 확인.

## 3. 수동 스모크 테스트 (15분)

1. **가입/온보딩** — 새 계정 → slug 설정 → `/dashboard/edit/{id}`
2. **다중 프로필** — 대시보드 → 프로필 추가 → 라벨 입력 → 두 번째 slug
3. **발행** — 이력서 최소 입력 → 발행하기 → `/@slug` 공개 확인
4. **재편집** — 발행 후 수정 → 대시보드 이탈 → 공개 프로필 여전히 노출
5. **채팅** — 비로그인 `/@slug` → 질문 1회 → 스트리밍 응답
6. **문의** — 공개 프로필 문의 폼 제출 → 대시보드 받은 질문 탭
7. **신고** — 공개 프로필 신고하기 (테스트 후 admin에서 확인)

자동화: `npm run test:e2e:integration` (GitHub Secrets 또는 `.env.local`에 `E2E_TEST_*` 필요)

## 4. Go / No-Go

| 조건 | Go |
|------|-----|
| Upstash + Gemini + Supabase secrets 설정 | ✅ |
| migration 14~16 원격 적용 | ✅ |
| 스모크 7단계 통과 | ✅ |
| Sentry DSN 설정 (권장) | ✅ |

위 4항목 충족 시 **소프트 런치(10~30명)** 진행.
