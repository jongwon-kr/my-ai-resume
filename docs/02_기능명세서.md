# CloneCV 기능 명세서 (Feature Specification)

버전: v1.0 (MVP) | 작성일: 2026-07-05

각 기능에 대해 화면 흐름, 입출력, 예외 처리까지 Cursor에게 그대로 전달 가능한 수준으로 상세화했습니다.

---

## F-01. 회원가입 / 로그인

**화면**: `/signup`, `/login`

| 항목 | 내용                                                                        |
| ---- | --------------------------------------------------------------------------- |
| 입력 | 이메일, 비밀번호 (or Google OAuth)                                          |
| 처리 | Supabase Auth 사용. 회원가입 시 `profiles` 테이블에 row 자동 생성 (trigger) |
| 출력 | 성공 시 `/onboarding`(고유 ID 설정)으로 리다이렉트                          |
| 예외 | 이메일 중복, 비밀번호 규칙 미달, OAuth 실패 → 토스트 에러 메시지            |

## F-02. 고유 ID(슬러그) 설정 (온보딩)

**화면**: `/onboarding`

| 항목 | 내용                                                           |
| ---- | -------------------------------------------------------------- |
| 입력 | 원하는 ID (영문/숫자/하이픈, 3~20자)                           |
| 처리 | 실시간 중복 체크 (debounce 500ms), `profiles.slug` UNIQUE 제약 |
| 출력 | 사용 가능 시 초록 체크, 불가 시 대안 3개 자동 제안             |
| 예외 | 예약어(admin, api 등) 차단 리스트 적용                         |

## F-03. 멀티스텝 이력서 입력 폼

**화면**: `/dashboard/edit` (9섹션 연속 스크롤)

| Step | 항목              | 필드                                                             |
| ---- | ----------------- | ---------------------------------------------------------------- |
| 1    | 기본 정보         | 이름, 직무, 한줄소개, 사진, 연락처, **외부 링크**, PII 표시 토글 |
| 2    | 경력              | 회사, 직책, 기간, 설명 (드래그 정렬)                             |
| 3    | 학력              | 학교, 전공, 학위, 상태, 기간                                     |
| 4    | 자격·어학·수상    | 카테고리, 항목명, 발급처, 취득일                                 |
| 5    | 경험/활동/교육    | 활동명, 기관, 기간, 설명                                         |
| 6    | 기술 스택         | 태그 + 숙련도, 드래그 정렬                                       |
| 7    | 프로젝트 (최대 3) | STAR + 트러블슈팅                                                |
| 8    | 자기소개서        | 제목, 본문                                                       |
| 9    | 예상 질문 답변    | Q&A, 챗 미리보기 패널                                            |

| 공통 처리   | 내용                                                                                            |
| ----------- | ----------------------------------------------------------------------------------------------- |
| 자동저장    | 각 필드 blur 시 또는 30초 간격 debounce 저장 (`status: draft`)                                  |
| 유효성 검사 | 필수 필드 미입력 시 다음 단계 이동 차단, 글자수 카운터 표시                                     |
| 완료 처리   | Step 4 완료 버튼 클릭 → `generate_system_prompt` 서버 함수 호출 → `profiles.status = published` |

## F-04. 시스템 프롬프트 생성 엔진 (백엔드)

| 항목   | 내용                                                                                                          |
| ------ | ------------------------------------------------------------------------------------------------------------- |
| 트리거 | 폼 제출 완료 or 수정 후 재발행                                                                                |
| 입력   | `profiles`, `projects` 테이블 데이터                                                                          |
| 처리   | 템플릿 기반으로 구조화된 프롬프트 조합 (아래 3번 아키텍처 문서의 프롬프트 템플릿 참고) + 가드레일 지시문 삽입 |
| 출력   | `system_prompts` 테이블에 최종 프롬프트 텍스트 저장 (버전 관리)                                               |
| 비고   | 매번 LLM에 다시 던지지 않고, 조합 로직은 서버 코드(문자열 템플릿)로 처리해 비용/지연 최소화                   |

## F-05. 퍼블릭 프로필 페이지

**화면**: `/@[id]` (SSR)

| 영역      | 내용                                                                               |
| --------- | ---------------------------------------------------------------------------------- |
| 좌측 패널 | 이력서 UI: 기본정보, 기술스택 뱃지, 프로젝트 카드 (아코디언으로 상세 펼침)         |
| 우측 패널 | 채팅 UI: 말풍선, 추천 질문 칩("가장 어려웠던 프로젝트는?" 등 자동 생성), 입력창    |
| 하단      | 워터마크 CTA: "이 이력서 봇, 어떻게 만들었을까요? 무료로 만들기 →"                 |
| SEO       | OG 이미지 동적 생성(이름/직무/기술스택 포함 카드), `<title>`, `<meta description>` |
| 모바일    | 탭 전환형 (이력서 탭 / 채팅 탭)                                                    |
| 비공개 시 | "비공개 프로필입니다" 안내 페이지로 대체                                           |

## F-06. AI 챗봇 대화

| 항목      | 내용                                                                                                                                                     |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 입력      | 방문자의 텍스트 질문                                                                                                                                     |
| 처리 흐름 | 1) 레이트리밋 체크 → 2) 질문 + 시스템 프롬프트 + 최근 대화 히스토리(최대 N턴)를 조합 → 3) LLM 스트리밍 호출 → 4) 응답 실시간 렌더 → 5) 대화 로그 DB 저장 |
| 가드레일  | 시스템 프롬프트 내 명시: "제공된 정보 밖의 질문에는 반드시 정해진 문구로 답한다" + 응답 후처리 필터(민감 키워드 감지 시 대체 문구 강제 치환)             |
| 예외      | LLM 오류/타임아웃 시 "잠시 후 다시 시도해주세요" 표시, 재시도 버튼                                                                                       |
| 대화 저장 | 소유자 대시보드에서 세션별 열람 가능, 방문자 식별 정보는 저장하지 않음(익명)                                                                             |

## F-07. 소유자 대시보드

**화면**: `/dashboard`

| 탭          | 내용                                            |
| ----------- | ----------------------------------------------- |
| 프로필 관리 | 수정, 비공개 전환, 링크 복사                    |
| 대화 로그   | 날짜별 세션 목록, 클릭 시 전체 대화 열람        |
| 통계        | 조회수, 대화 세션 수, 최근 7일 추이 (간단 차트) |
